# Update: Neural Reflex Test Latency Fixes
**Timestamp:** 2026-03-28

## 🚀 Resolved 900ms Reaction Time Discrepancy
The issue where users clicking in 200ms were seeing 1100ms outputs has been resolved through the following optimizations:

### 1. Synchronous State Management
- Moved `stateRef` updates and `startTime` capture to `useLayoutEffect`.
- This ensures the internal timing reference is updated *before* the browser paints the "READY" signal, eliminating the 100-300ms "post-paint" delay caused by standard `useEffect`.

### 2. MediaPipe "Warm-Up" Phase
- Modified the processing loop to start sending camera frames to MediaPipe during the "WAITING" (Get Ready) phase.
- This eliminates the "cold start" latency of the AI model, ensuring it is already primed and detecting landmarks the instant the test becomes active.

### 3. Increased Sampling Resolution
- Boosted camera frame rate from 15 FPS to 30 FPS.
- This reduces the maximum potential sampling error from ~66ms to ~33ms, providing more granular and accurate reaction time data.

### 4. Precision Timing Logic
- Refined the `onResults` and `handleMotorClick` handlers to use `performance.now()` for sub-millisecond precision.
- Removed arbitrary latency floors (e.g., the 200ms minimum) to allow for authentic, high-performance user results.

## 📊 Implemented Relative Testing Thresholds
To prevent false positives caused by hardware/browser latency, the risk assessment engine has been tuned for realistic home-use:
- **Baseline Adjustment:** Raised Motor baseline to 450ms and Facial baseline to 500ms to account for webcam/MediaPipe overhead.
- **Dynamic Risk Scaling:** Adjusted the Sigmoid models in the backend to ensure a 400ms reaction time is classified as "Low Risk" (~15-25%), only flagging high risks when latencies exceed 700-850ms.
- **Dashboard Synchronization:** Updated the frontend Results Dashboard to reflect these new baselines, providing consistent feedback between the live test and the final report.

## 💾 Neon DB Integration & ALS Pivot
- **Persistence Layer:** Integrated Neon (PostgreSQL) with `asyncpg`. Automatically synchronizes user identity and persists raw biomarkers + computed risks.
- **ALS Risk Assessment:** Pivoted from cognitive decline to ALS, focusing on Bulbar onset (Speech/Jitter) and Neuromotor execution (Latency).
- **Longitudinal Readiness:** Schemas designed to support 5-test moving averages and trend analysis for progressive neurological conditions.

## 🏗️ Frontend Architecture Modularization
- **Modular Refactoring:** Decomposed the monolithic `DEMO1.js` into a clean, reusable structure:
    - `src/components/UI/`: Base design primitives (GlassContainer, AmbientBackground).
    - `src/components/`: Core features (Dashboard, TestModules).
    - `src/pages/`: View-level orchestration (Landing, TestPage).
    - `src/lib/`: Unified API client and risk calculation logic.
- **Maintainability:** Improved code readability and separated concerns, allowing for independent scaling of test modules and UI components.

**Status:** Performance stabilized, DB integrated, and architecture modernized for future expansion.
